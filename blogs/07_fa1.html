Transformers

Large language models(LLM) in general use transformers based architecture and attention forms the heart of any transformer.

Attention says given three matrices Q, K and V of size (N, d), we perform self attention using

S = Q . K^T

P = softmax(S)

O = P.V

S is whats called the attention score, P when it goes through softmax, we get normalized attention scores and finally O is final output after a linear transformation. O is going to be of size seq len * dim( technically this is head dim but lets worrry about it later)

well, from this , the most intensive operation has to be computing S and O. they involve matrix multiplications but even worse, they have to be done sequentially. the complexity for this operation in general is O(N2), (explain why)

What is flash attention doing then?

whats the simplest way you can think of making this fast? hmmm using them in lower precision? or using sparse matrices, great ideas but they are all approximations but what we need is the exact answer self attention gives us.

Compute bound vs memory bound

sorry last concept before we move into flash attention, hehe

compute bound is where the device spends majority of its time on computation. data flow is faster than the gpu can complete its operations

memory bound is where the device is actually stalled moving things rather than the computation
